MÓDULO 05 · LAYOUT FLEX

Layout con flexbox: el split principal

La herramienta divide la pantalla en dos columnas: el formulario (izquierda) y el output (derecha). Toda la barra superior es fija. Esto se logra con flexbox y un truco poco conocido: min-height: 0.

🎨 CSS~10 minIntermedio
1La barra superior: flex 0 0 auto

El body es un flex column. La barra superior (.top-tabs) no debe crecer ni encogerse: ocupa su altura y ya. El área principal (#subTabsAndMain) absorbe todo el resto.

CSS
/* La barra superior: NO crece, NO encoge */
.top-tabs {
  display: flex;
  flex: 0 0 auto;   /* grow=0, shrink=0, basis=auto */
  height: 34px;
}

/* El área principal: absorbe todo el espacio restante */
#subTabsAndMain {
  flex: 1;          /* grow=1 */
  min-height: 0;    /* EL TRUCO clave (ver sección 3) */
}
flex: 0 0 auto = "soy rígido, mi tamaño es mi contenido". flex: 1 = "soy elástico, absorbo lo que sobre".
2El split horizontal: form-col + output-panel

Dentro de #subTabsAndMain hay un .main que es un flex row. A la izquierda el formulario, a la derecha el output:

CSS
.main {
  display: flex;      /* row por defecto */
  flex: 1;
  min-height: 0;
}
.form-col {
  width: 50%;
  min-width: 380px;   /* mínimo para que sea usable */
  display: flex;
  flex-direction: column;
  border-right: 1px solid #d1d5db;
  overflow: hidden;
}
.output-panel {
  flex: 1;             /* absorbe el resto */
  display: flex;
  flex-direction: column;
  background: #fafbfc;
  overflow: hidden;
}

¿Por qué form-col usa width y no flex?

Si .form-col usara flex: 1 y .output-panel también flex: 1, cada uno ocuparía 50% exactos. Pero la herramienta quiere que el formulario tenga un ancho fijo de 50% con un mínimo de 380px, y el output absorba el resto. width: 50% + min-width: 380px lo logra: si la ventana es angosta, el formulario no se encoge menos de 380px.

3El truco de min-height: 0

El problema que todos encontramos

Tienes un contenedor flex vertical con un hijo que hace scroll. Le pones overflow-y: auto al hijo. Y no funciona: el contenido se desborda y aparece scroll en la página entera.

¿Por qué? Por defecto, un flex item tiene min-height: auto: no puede ser más pequeño que su contenido. Así que cuando el contenido crece, el hijo empuja al padre hacia abajo.

La solución: min-height: 0 en el flex item. Esto anula el auto por defecto y permite que el hijo sea más pequeño que su contenido. Entonces overflow-y: auto funciona y aparece el scroll dentro del hijo.
El patrón en la herramienta
/* El body es flex-column con overflow:hidden */
body { display: flex; flex-direction: column; overflow: hidden; }

/* subTabsAndMain: flex:1 + min-height:0 (¡sin esto no hay scroll!) */
#subTabsAndMain { flex: 1; min-height: 0; }

/* main: otro flex:1 + min-height:0 */
.main { flex: 1; min-height: 0; }

/* form-col: flex-column, scrollbar aquí */
.form-col { display: flex; flex-direction: column; min-height: 0; }

/* Y el panel que hace scroll: */
.form-panel { flex: 1; overflow-y: auto; min-height: 0; }

Es un patrón repetido: cada contenedor intermedio tiene min-height: 0. Es molesto pero necesario con flexbox. CSS Grid no tiene este problema (por eso la herramienta usa grid para los colapsables, en el siguiente módulo).
