MÓDULO 04 · BASE CSS

Reset, base y tokens de la herramienta

La herramienta usa un tema claro (fondo gris claro, botones blancos con bordes). Antes de posicionar nada, hay un reset que iguala los navegadores y estilos base para todos los controles. Vamos a ver cómo empieza el CSS.

🎨 CSS ~10 min de lectura Nivel: principiante
1El reset universal

La primera línea de CSS

El CSS de la herramienta empieza con la línea más conocida del mundo CSS:

CSS
*, *::before, *::after {
  margin: 0;
  padding: 0;
  box-sizing: border-box;
}

¿Por qué box-sizing: border-box?

Sin box-sizing: border-box, el navegador calcula el ancho de un elemento sumando width + padding + border. Si dices width: 100px y le pones padding: 10px y border: 1px, el elemento ocupa 122px. Con border-box, el width incluye padding y border: ocupa exactamente 100px. Esto evita que los layouts se rompan cuando añades bordes o padding.

2El body como contenedor flex
CSS
html, body { height: 100%; }
body {
  font-family: -apple-system, BlinkMacSystemFont, "Segoe UI", sans-serif;
  font-size: 13px;
  color: #1d2129;
  background: #f5f6f7;
  overflow: hidden;
  display: flex;
  flex-direction: column;
}

¿Por qué overflow: hidden?

La herramienta no quiere que la página entera se desplace. Cada panel (formulario, output) tiene su propio scroll interno. overflow: hidden en el body asegura que ningún contenido rompa la altura fijada: los hijos quienes gestionan su propio scroll.

font-family con system fonts

La herramienta no carga Google Fonts (restricción cero-red). Usa -apple-system, BlinkMacSystemFont, "Segoe UI": las tipografías nativas de cada sistema operativo. En Mac se ve San Francisco, en Windows se ve Segoe UI. Sin descargar nada.

3Estilos base de botones e inputs

En la herramienta, todos los botones comparten el mismo estilo base. Luego, clases específicas (.primary, .danger) añaden color:

CSS
button {
  font-family: inherit;
  font-size: 12px;
  cursor: pointer;
  border: 1px solid #c7ccd1;
  background: #fff;
  padding: 4px 10px;
  border-radius: 3px;
  color: #1d2129;
}
button:hover { background: #f0f2f5; }
button:active { background: #e4e6eb; }
button:disabled { cursor: not-allowed; opacity: 0.5; }

/* Variante azul para acciones principales */
button.primary {
  background: #2563eb;
  color: #fff;
  border-color: #1d4ed8;
}

/* Variante roja para acciones destructivas */
button.danger {
  color: #b91c1c;
  border-color: #fca5a5;
}
4Focus states: la validación visual

Cuando enfocas un input, el navegador da un outline por defecto. La herramienta lo sustituye por un borde azul y un halo de sombra:

CSS
input:focus, textarea:focus, select:focus {
  outline: none;
  border-color: #2563eb;
  box-shadow: 0 0 0 2px rgba(37, 99, 235, 0.2);
}

/* Cuando el campo tiene error: borde rojo + halo rojo */
.field.invalid input,
.field.invalid textarea {
  border-color: #dc2626;
  box-shadow: 0 0 0 2px rgba(220, 38, 38, 0.18);
}

El truco del box-shadow como halo

Cuando un campo es inválido, en vez de cambiar el color de fondo (lo que mueve el layout), se usa box-shadow con 0 0 0 2px: un anillo del color del borde que no ocupa espacio. El border cambia a rojo y el halo rojo lo rodea. El usuario lo nota al instante sin que nada salte.
