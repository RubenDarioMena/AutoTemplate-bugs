MÓDULO 01 · HTML — ESTRUCTURA

El esqueleto HTML: contenedores y layout

La herramienta vive en un solo archivo .html. Antes de que haya colores, animaciones o JavaScript, hay HTML: contenedores vacíos que definen qué zonas existen y cómo se organizan. En este módulo abrimos el archivo y miramos el esqueleto: la barra superior, el área principal, los paneles laterales y los contenedores ocultos.

~12 min de lectura Necesitas bug_tool.html Bloque I · HTML
1La idea en una imagen

Imagina la herramienta como un taller con varias mesas de trabajo. Cada mesa tiene un propósito: una para escribir el bug, otra para ver el resultado, otra para editar datos. En HTML, cada mesa es un <div> — un contenedor vacío que espera a que CSS le dé forma y JS lo llene de contenido.

Barra superior
Las pestañas Bug, Regression, Data y Rules. Es el techo del taller.
Área principal
Dos columnas: formulario a la izquierda, output a la derecha.
Paneles ocultos
Data, Rules y el modal están en el HTML pero escondidos con display:none.
2El <head>: metadatos y estilos

Lo primero que lee el navegador

El <head> no se ve en pantalla, pero configura todo lo que el navegador necesita antes de pintar un solo pixel. En la herramienta tiene tres cosas:

HTML
<!-- El head configura el documento -->
<head>
  <meta charset="UTF-8" />          <!-- caracteres Unicode (acentos, ñ) -->
  <meta name="viewport" content="width=device-width, initial-scale=1.0" />  <!-- responsive -->
  <title>Bug Report Tool</title>       <!-- texto de la pestaña -->
  <style> ... </style>              <!-- TODO el CSS va aquí dentro -->
</head>

Los tres metas que importan

1

charset="UTF-8"

Permite usar acentos, ñ y caracteres especiales sin que aparezcan símbolos raros.

2

viewport

Le dice al navegador que adapte el ancho al dispositivo. Sin esto, en móvil se vería miniatura.

3

<style> embebido

El CSS va DENTRO del HTML, no en un archivo separado. Es lo que hace que la herramienta sea un solo archivo.

3El <body>: contenedores principales

Cinco zonas visibles

El <body> de la herramienta se divide en zonas. Cada zona es un <div> con un id o una class para que CSS y JS puedan encontrarla. De arriba abajo:

HTML — estructura del body
<body>

  <!-- 1. BARRA SUPERIOR: pestañas + botones de export -->
  <div class="top-tabs" id="topTabs">
    <div id="formTabs"></div>  <-- pestañas Bug/Regression (JS las llena) -->
    <div data-tab="data">Data</div>
    <div data-tab="rules">Rules</div>
    <div class="top-right">...</div>  <-- botones export, sesión -->
  </div>

  <!-- 2. BANNER de notificaciones (oculto por defecto) -->
  <div class="banner hidden" id="banner"></div>

  <!-- 3. BARRA DE TILES (chips de validación) -->
  <div class="tiles-bar" id="tilesBar"></div>

  <!-- 4. ÁREA PRINCIPAL: sub-pestañas + dos columnas -->
  <div id="subTabsAndMain">
    <div class="sub-tabs" id="subTabs"></div>
    <div class="main" id="main">
      <div class="form-col">        <-- columna izquierda -->
        <div class="form-toolbar">...</div>
        <div class="form-panel" id="formPanel"></div>
      </div>
      <div class="output-panel">     <-- columna derecha -->
        <textarea id="outputTextarea"></textarea>
        <div id="mediaPanel">...</div>  <-- nombres de media -->
      </div>
    </div>
  </div>
</body>

Anatomía de un contenedor

Cada <div> del body tiene dos formas de identificarse:

1

id — único en toda la página

JS lo usa con document.getElementById("formPanel"). Solo puede haber UN elemento con cada id.

2

class — compartible

CSS lo usa con .top-tabs { ... }. Varios elementos pueden tener la misma class.

4Contenedores ocultos: Data, Rules y el modal

Existen en el HTML pero no se ven

La herramienta tiene tres paneles que no se muestran al abrir el archivo: la pestaña Data, la pestaña Rules y el modal de confirmación. Están en el HTML desde el principio, pero con style="display:none" el navegador no los pinta hasta que JS los activa.

HTML — paneles ocultos
<!-- Pestaña Data: edita listas y reglas -->
<div id="dataContainer" style="display:none;">
  <div id="dataSidebar"></div>
  <div id="dataTableWrap"></div>
</div>

<!-- Pestaña Rules: condiciones y tiles -->
<div id="rulesContainer" style="display:none;"></div>

<!-- Modal: diálogo de confirmación -->
<div class="modal-backdrop" id="modal">
  <div class="modal" id="modalBox">
    <h3></h3>
    <div class="modal-body" id="modalBody"></div>
    <div id="modalActions"></div>
  </div>
</div>

Inputs de archivo invisibles

La herramienta necesita botones que abran el diálogo de "seleccionar archivo" del sistema operativo. El navegador solo permite esto con <input type="file">, así que se crean ocultos y JS los activa con un click programado:

HTML — inputs ocultos
<input type="file" id="dataCsvInput"
       accept=".csv" style="display:none" />

<input type="file" id="sessionFileInput"
       accept=".json" style="display:none" />
5Tu primer momento: identifica los contenedores

Abre un editor de texto y escribe los IDs de los contenedores que encuentres en bug_tool.html. Este código simula la búsqueda — pulsa Ejecutar para ver qué IDs existen en el body:

JavaScript — buscar contenedores por id 
// Lista de IDs que el JavaScript busca con getElementById:
var contenedores = [
  "topTabs", "formPanel", "outputTextarea",
  "tilesBar", "modal", "banner",
  "dataContainer", "rulesContainer",
  "subTabs", "mediaPanel", "formTitle"
];

contenedores.forEach(function(id) {
  var el = document.getElementById(id);
  console.log(id + ": " + (el ? "✓ existe" : "✗ no encontrado"));
});

6Ejercicio: reconstruir el esqueleto

Crea un HTML con 4 de los contenedores

Sin mirar el archivo original, crea un .html nuevo con esta estructura:

1

Un <div id="topTabs">

Dentro, dos <div> con texto "Bug" y "Data".

2

Un <div id="main">

Dentro, dos <div>: uno con class "form-col" y otro con class "output-panel".

3

Un <div id="modal" style="display:none">

Dentro, un <h3> vacío y un <div id="modalBody">.

4

Ábrelo en el navegador

No verás ninguna barra con pestañas (aún no hay CSS). Solo texto sin formato. Eso es normal: el HTML es la estructura, no el estilo.
