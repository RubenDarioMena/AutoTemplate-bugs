MÓDULO 10 · CONFIG

Cargar y migrar la configuración

La herramienta mantiene dos copias de la configuración: vanilla (lo que trae el archivo) y config (lo que el usuario ha modificado). Al cargar, decide cuál usar. Y si el archivo trae listas nuevas que la config guardada no conoce, las completa sin pisar nada.

⚡ JS~10 minIntermedio
1loadVanillaConfig: leer el JSON del HTML
JavaScript
function loadVanillaConfig() {
  const el = $("vanillaConfig");
  vanilla = JSON.parse(el.textContent);
}

Esta función lee el bloque <script type="application/json"> con textContent y lo convierte en un objeto JavaScript con JSON.parse. Ese objeto es la configuración vanilla: la que viene dentro del .html tal como se distribuye.

2loadConfig: localStorage o clonar vanilla
JavaScript
function loadConfig() {
  try {
    const raw = localStorage.getItem(CONFIG_KEY);
    if (raw) {
      const c = JSON.parse(raw);
      if (c && c.schema && c.data) {
        config = c;
        migrateConfig();
        return;
      }
    }
  } catch (e) { console.error("loadConfig", e); }
  // Si no hay config guardada o está corrupta, usar vanilla:
  config = deepClone(vanilla);
}

El patrón vanilla → copia local

1. Intenta leer de localStorage (lo que el usuario guardó la última vez).

2. Si existe y es válido (tiene schema y data), lo usa y llama a migrateConfig.

3. Si no existe o está corrupta, hace una copia profunda de vanilla con deepClone. Nunca usa vanilla directamente: si lo hiciera, cualquier cambio mutaría el original.

3migrateConfig: compatibilidad hacia atrás

Imagina que el equipo de Database añade una lista nueva ("weather") al archivo bug_tool.html que se distribuye. Pero el tester tiene una config guardada en su navegador que no tiene esa lista. migrateConfig completa lo que falta sin pisar lo que el usuario ya cambió:

JavaScript
function migrateConfig() {
  // Para cada lista de vanilla que no exista en config, añadirla:
  Object.entries(vanilla.data.lists).forEach(([k, v]) => {
    if (!config.data.lists[k]) config.data.lists[k] = deepClone(v);
  });
  // Igual con las reglas:
  Object.entries(vanilla.data.rules).forEach(([k, v]) => {
    if (config.data.rules[k] === undefined)
      config.data.rules[k] = v;
  });
}
La idea clave: nunca reemplazar, solo añadir lo que falta. Si el usuario borró una lista a propósito, migrateConfig no la revive. Solo añade listas/reglas nuevas que el usuario nunca tuvo.
4configIsDirty: detectar cambios locales
JavaScript
function configIsDirty() {
  return JSON.stringify(config) !== JSON.stringify(vanilla);
}

function updateConfigChip() {
  $("configChip").classList.toggle("visible", configIsDirty());
}

Comparar dos objetos en JavaScript no se puede con === (compara referencias, no contenido). La técnica usada: serializar ambos a texto y comparar los strings. Si son distintos, se muestra el aviso "Config modificada" en la barra superior, recordando al usuario que debe exportar para compartir sus cambios.
