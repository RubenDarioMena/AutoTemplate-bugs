Funciones utility que se usan en todo

La herramienta tiene un grupo de funciones pequeñas que se llaman desde todas partes: escapar HTML, clonar objetos, convertir texto a id seguro, descargar archivos, copiar al portapapeles. Son el pegamento del código.

1El atajo $()
JavaScript
function $(id) { return document.getElementById(id); }

En la herramienta se llama a getElementById cientos de veces. Esta función de 1 línea ahorra teclear el nombre largo. En vez de document.getElementById("formPanel"), escribes $( "formPanel"). El símbolo $ es un nombre de función válido en JavaScript.

2escapeHtml: nunca confíes en el input
JavaScript
function escapeHtml(s) {
  return String(s == null ? "" : s)
    .replace(/&/g, "&amp;")
    .replace(/</g, "&lt;")
    .replace(/>/g, "&gt;")
    .replace(/"/g, "&quot;");
}
¿Por qué es crítico? Si un usuario escribe <script>alert('hack')</script> en un campo y la herramienta lo inserta con innerHTML sin escapar, se ejecuta el script. Escapando <, >, & y ", el texto se muestra literal sin interpretarse como HTML.
3deepClone: copia vs referencia
JavaScript
function deepClone(obj) {
  return JSON.parse(JSON.stringify(obj));
}

En JavaScript, los objetos se pasan por referencia: si haces const copy = obj y mutas copy.values, también mutas obj.values. Para trabajar con la herramienta necesitamos una copia independiente del vanilla, así poder modificarla sin tocar el original. JSON.stringify → JSON.parse hace una copia profunda: todo se serializa a texto y vuelve como objeto nuevo.

4slugify y resolveRef
JavaScript
// "Posición de Vista" → "posicionDeVista"
function slugify(label) {
  const clean = String(label || "")
    .normalize("NFD").replace(/[\u0300-\u036f]/g, "")  // sin acentos
    .replace(/[^a-zA-Z0-9]+/g, " ").trim();
  // camelCase: primera palabra en minúscula, el resto Capitalizada
  const parts = clean.split(" ");
  return parts[0].toLowerCase() +
    parts.slice(1).map(p => p[0].toUpperCase() + p.slice(1).toLowerCase()).join("");
}

// ResolveRef: "@coord_re" → busca coord_re en config.data.rules
function resolveRef(v) {
  if (typeof v === "string" && v.startsWith("@"))
    return config.data.rules[v.slice(1)] || "";
  return v == null ? "" : v;
}
5downloadFile y copyToClipboard
JavaScript
function downloadFile(name, content) {
  const blob = new Blob([content], { type: "text/plain" });
  const url = URL.createObjectURL(blob);
  const a = document.createElement("a");
  a.href = url; a.download = name;
  document.body.appendChild(a); a.click();
  document.body.removeChild(a);
  setTimeout(() => URL.revokeObjectURL(url), 1000);
}

function copyToClipboard(text) {
  navigator.clipboard.writeText(text).then(() => {
    flashBanner("Copiado al portapapeles.");
  }).catch(() => {
    // Fallback para navegadores sin Clipboard API
    const ta = document.createElement("textarea");
    ta.value = text;
    document.body.appendChild(ta); ta.select();
    document.execCommand("copy");
    document.body.removeChild(ta);
  });
}

downloadFile crea un Blob (datos en memoria), genera una URL temporal con createObjectURL, crea un enlace <a> invisible, simula un clic y lo limpia. copyToClipboard usa la Clipboard API moderna con un fallback a execCommand("copy") para navegadores antiguos.
