MÓDULO 08 · VARIABLES

Variables y constantes de la aplicación

La primera sección de JavaScript de la herramienta declara todas las variables y constantes globales. Algunas no cambian nunca (constantes), otras se reasignan (let). Vamos a ver cada una y por qué es let o const.

⚡ JS~10 minPrincipiante
1PRISTINE_HTML: capturar el HTML intacto
JavaScript
// Lo primero que ejecuta la herramienta:
const PRISTINE_HTML = "<!DOCTYPE html>\\n" + document.documentElement.outerHTML;

¿Por qué capturar el HTML al cargar?

La función "Exportar herramienta" descarga un nuevo bug_tool.html con la configuración modificada. Para hacer eso, necesita el HTML original —sin que el JavaScript haya modificado el DOM. Si esperara, el innerHTML de los paneles ya estaría lleno de formularios y campos. Capturándolo al principio (antes de que boot() renderice nada), tiene una copia limpia.

El "truco de los dos partes" ("") evita que el parser HTML corte el script aquí al ver </script>.

2CONFIG_KEY y SESSION_KEY: localStorage
JavaScript
const CONFIG_KEY  = "bug_tool_v2_config";   // schema + data
const SESSION_KEY = "bug_tool_v2_session";  // instancias + preferencias

Dos claves, dos propósitos

CONFIG_KEY guarda el schema y los datos (campos, listas, reglas). Cuando alguien edita algo en Data o Rules, se guarda aquí. Al "Exportar herramienta", este JSON se incrusta en el nuevo HTML.

SESSION_KEY guarda las instancias abiertas (Bug 1, Bug 2...) y preferencias (pestaña activa, colapso de tiles). Así, al cerrar y reabrir, el navegador recupera dónde estabas.

3El objeto state: el estado mutable global
JavaScript
const state = {
  topTab: "",            // id del formulario activo, "data" o "rules"
  editMode: false,
  instances: [],         // array de bugs abiertos
  activeInstanceId: null,
  dataTab: { selectedCategory: "" },
  prefs: { dontAskOnClose: false }
};
¿Por qué const y no let? Porque const no permite reasignar la variable, pero sí permite mutar su contenido. No puedes hacer state = {}, pero sí state.instances.push(...) o state.topTab = "bug". Es lo que queremos: el objeto siempre es el mismo, pero sus propiedades cambian.

5Tu primer momento

Pulsa Ejecutar para ver las variables en acción. El código declara una versión simplificada del estado y lo muestra.

JavaScript
const CONFIG_KEY = "bug_tool_v2_config";
const state = { topTab: "bug", instances: [], editMode: false };

console.log("CONFIG_KEY:", CONFIG_KEY);
console.log("state inicial:", state);

// Mutar el objeto (const permite mutar, no reasignar)
state.instances.push({ id: "bug_1", title: "Bug 1" });
state.topTab = "bug";

console.log("state después de mutar:", state);
Salida de la consola
